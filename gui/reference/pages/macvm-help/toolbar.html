Toolbar Reference

The eleven buttons across the top of every page, left to right, and what each one does today:

  • Back — go to the previous page in this window's local history. Wired up.
  • Forward — go to the next page in this window's local history. Wired up.
  • Start page — return to startPage.html. Wired up.
  • Find definition — look up a class or method definition. Wired to the VM's browser: it resolves against the real object world.
  • Implementors — list the methods that implement a given message selector. Wired to the VM's browser.
  • Senders — list the methods that send a given selector. Wired to the VM's browser.
  • User hierarchy — browse the class hierarchy rooted at user-defined classes. Wired to the VM's browser.
  • Full hierarchy — browse the complete class hierarchy. Wired to the VM's browser.
  • Workspace — open a blank scratch page for evaluating expressions. Backed by the embedded VM — the expressions you type run for real.
  • Text editor — open the built-in text editor. Not wired yet.
  • Documentation — open this help viewer. Wired up.

The class-lookup and browser buttons are now backed by MACVM's real object world through the embedded VM (see MACVM architecture): they look up, list, and browse against the live class dictionary rather than a stub. Better still, the browser is a full editor — a method you change is live-compiled into the running world and persisted to the SQLite image store, so the edit survives a restart.
