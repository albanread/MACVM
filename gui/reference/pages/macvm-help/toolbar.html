Toolbar Reference

The eleven buttons across the top of every page, left to right, and what each one does today:

  • Back — go to the previous page in this window's local history. Wired up.
  • Forward — go to the next page in this window's local history. Wired up.
  • Start page — return to startPage.html. Wired up.
  • Find definition — look up a class or method definition. Not wired yet — needs a running VM to look anything up in.
  • Implementors — list the methods that implement a given message selector. Not wired yet, same reason.
  • Senders — list the methods that send a given selector. Not wired yet, same reason.
  • User hierarchy — browse the class hierarchy rooted at user-defined classes. Not wired yet.
  • Full hierarchy — browse the complete class hierarchy. Not wired yet.
  • Workspace — open a blank scratch page for evaluating expressions. Not wired yet.
  • Text editor — open the built-in text editor. Not wired yet.
  • Documentation — open this help viewer. Wired up.

"Not wired yet" buttons still log a note to the transcript when clicked, so clicking them is safe to try — they just don't do anything useful until MACVM has a VM behind the shell (see MACVM architecture) to actually look up classes, methods, and senders against.
