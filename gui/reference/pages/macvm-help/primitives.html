Predefined Primitives

A primitive is a VM operation a method can run before its Smalltalk body. A method marked <primitive: N> tries primitive N first; if the primitive succeeds, the method returns its result immediately, and if it fails, execution falls through to the ordinary Smalltalk code below — which is how, for example, SmallInteger + handles overflow by promoting to a large integer. There are 153 numbered primitives; the tables below group them by area.

The columns: No. is the primitive number written in the pragma; Selector is the message it implements; Args is the argument count excluding the receiver; and Notes record whether it may allocate (and so reach the garbage collector) and whether it can fail into the Smalltalk fallback. The optimizing compiler shims most of these directly into native code.

Multi-VM workers

Spawn and talk to worker VMs — each its own heap, JIT, and GC on its own OS thread. Everything that crosses a VM boundary is a deep copy carried as MOP pickle bytes; no oop is ever visible to two VMs. All fail cleanly in a VM with no worker role. See world/47_worker.mst and docs/multi-smalltalk-worker.md.

Reflection

The three questions a program can ask about itself that Smalltalk cannot answer in Smalltalk. copy is not here: it is ordinary Smalltalk (shallowCopy then postCopy) in world/59_reflection.mst — the second step matters, because a shallow copy shares whatever its slots point at, so collections override postCopy to un-share their internal storage.

The FFI primitive

One primitive is not numbered in the table above: an <primitive: FFI …> pragma marks a method as a foreign call. It carries a sentinel id (-1) and a descriptor of the C function and its argument shape rather than a fixed number, and dispatches through the FFI trampolines. See docs/FFI.md.
