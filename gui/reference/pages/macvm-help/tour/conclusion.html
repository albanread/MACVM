Conclusion

MACVM is a working research virtual machine. The core is done: the object world, the dispatch-based bytecode interpreter, generational scavenge plus a full compacting collector running underneath live, moving compiled frames, and the adaptive optimizing compiler — inline caches, customization, method and block inlining, deoptimization, on-stack replacement. Around that core sit a Tier-1 POSIX FFI, GUI embedding through a crash-surviving VmHandle, a SQLite image store with a byte-identical DB → VM boot loader, and full closure compilation — literal blocks that splice inline rather than allocate.

Two theses are settled. Compiler coverage is achieved — almost everything that runs, runs compiled, and stays compiled. And the reliability thesis is realized: the VM is fast and stable at the same time, which is exactly the combination the released Strongtalk could never hold. deltablue runs about 53x faster under the compiler than under the interpreter, and a scavenge or a full compaction can run underneath that compiled code without disturbing it. Fast was never the hard part. Fast-and-stable is, and that is the part that works.

Still open

  • Tier-2 Cocoa FFI — doesNotUnderstand:-driven message dispatch out to native Cocoa objects.

  • An in-image tool library — mirrors, browsers, and an inspector written in Smalltalk itself rather than in the Rust host.

  • Live smappl widgets — today the tag renders an inert placeholder box naming its code; wiring it to real evaluation is future work.

  • Green threads with async C callouts — concurrency the FFI does not yet offer.

  • An optional static type system — Strongtalk's, if it is ever built here.

The released Strongtalk was a brilliant ten-minute demo that could not be made to stand up. Its ideas were right; the artifact could not carry them. MACVM is the wager that, with modern tooling and relentless differential testing, those same ideas can be made to hold still long enough to be useful — that you can have the adaptive-optimization dream and a system that does not fall over. It is still research, and the list above is real. But the thing runs, it runs fast, and it keeps running. That is the whole point, and it is the lesson Strongtalk taught most clearly, honored at last: Sign off.
