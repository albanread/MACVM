The Bytecode Set

A MACVM method is compiled to a compact stack-machine bytecode. The interpreter runs it with a fetch-decode-match loop — it reads the opcode byte and its operands inline — and the optimizing compiler decodes the same bytes into a control-flow graph. There are just 31 instructions. The opcode values below are the external wire format and are never renumbered.

A few conventions: an <i8> operand is a signed byte; <u8> an unsigned byte; <u16> a little-endian unsigned 16-bit value. “The stack” is the current activation's operand stack, and slot/index operands are small integers into the method's temps, literals, or the receiver's fields. The Bytes column is the total instruction length including operands.

Pushing values onto the stack

InstructionValueOperandsBytesEffect
push_self0x00—1Push the receiver, self.
push_nil0x01—1Push nil.
push_true0x02—1Push true.
push_false0x03—1Push false.
push_smi0x04<i8>2Push a small integer literal in -128..127.
push_literal0x05<u8 index>2Push literal[index] from the method's literal frame.
push_literal_w0x12<u16 index>3Wide push_literal — for a literal index ≥ 256.
push_temp0x06<u8 slot>2Push a temporary or argument slot (the two share one index space).
push_instvar0x09<u8 index>2Push an instance variable of the receiver.
push_global0x0B<u8 lit>2Push a global's value (the literal is its Association).
push_ctx_temp0x0F<u8 depth> <u8 idx>3Push a captured temp from an enclosing Context at depth.
push_closure0x11<u8 lit> <u8 ncopied>3Build a BlockClosure from a block literal, copying ncopied outer values.

Storing values

InstructionValueOperandsBytesEffect
store_temp0x07<u8 slot>2Store the top of stack into a temp; leaves it on the stack.
store_temp_pop0x08<u8 slot>2Pop the top of stack into a temp.
store_instvar_pop0x0A<u8 index>2Pop into an instance variable (with the GC write barrier).
store_global_pop0x0C<u8 lit>2Pop into a global's Association value.
store_ctx_temp_pop0x10<u8 depth> <u8 idx>3Pop into a captured Context temp at depth.

Stack manipulation

InstructionValueOperandsBytesEffect
pop0x0D—1Discard the top of stack.
dup0x0E—1Duplicate the top of stack.

Message sends

InstructionValueOperandsBytesEffect
send0x20<u8 ic>2Send a message. ic indexes the send-site / inline-cache table, which carries the selector and argument count.
send_super0x21<u8 ic>2A super send — lookup starts above the sending method's class.
send_w0x22<u16 ic>3Wide send — for a send-site index ≥ 256.
send_super_w0x23<u16 ic>3Wide super send.

Jumps and branches

InstructionValueOperandsBytesEffect
jump_fwd0x30<u16 dist>3Unconditional forward jump.
jump_back0x31<u16 dist>3Unconditional backward jump (a loop back-edge; where the JIT counts loop trips and may enter compiled code).
br_true_fwd0x32<u16 dist>3Pop the top of stack; branch forward if it is true (a non-boolean is an error).
br_false_fwd0x33<u16 dist>3Pop the top of stack; branch forward if it is false.

Returns

InstructionValueOperandsBytesEffect
return_tos0x40—1Return the top of stack from the method.
return_self0x41—1Return self.
block_return_tos0x42—1Return the top of stack from a block, to the block's caller.
nlr_tos0x43—1Non-local return: return the top of stack from the block's home method.

Notes

  • Temps and arguments share one index space (push_temp), so argument 0 is simply temp 0.

  • A send carries no selector inline — the ic operand names a send site, whose inline cache remembers the selector, the argument count, and the receiver classes seen so far. That cache is the type feedback the optimizing compiler reads.

  • push_closure, push_ctx_temp, block_return_tos, and nlr_tos are the block and closure instructions; the compiler splices most blocks inline rather than building a real closure (see the tour's Optimizing Compiler page).
