About this GUI Shell

MACVM's GUI is a native Cocoa window (NSWindow) holding a single WKWebView. Only the menu bar at the top of the screen is native; the toolbar and status bar you see inside the window are HTML, injected into every page when it loads. This whole page, in fact, was written as a plain .html file and had that chrome added the same way.

The toolbar and status bar

The row of icon buttons at the top and the bar at the very bottom are not part of any Strongtalk page's own markup — they are added at load time, so every page gets them for free. See the toolbar reference for what each button does.

The bar along the bottom shows the target of whatever link or button your pointer is currently over, and says "Ready" the rest of the time — hover over the link below to see it change: Documentation Index

doit= links

An ordinary Strongtalk page can write <a doit="some Smalltalk code">label</a> instead of a normal href. Clicking it doesn't navigate anywhere — it runs the code. Today, with no VM wired up yet, "running the code" means the shell echoes the code's source text into the transcript pane, so the click round trip (page → JavaScript → native host → transcript) can be built and tested well before there is a VM to actually execute it against. The smappl tag

A Strongtalk page can also write <smappl visual="some Smalltalk code"> (note: no closing tag) to embed a live widget — a class hierarchy browser, say — directly inline in the page, built by evaluating the given expression. Theme

The native Theme menu in the menu bar switches between Classic (the period-accurate Strongtalk look this page is shown in right now — pixel icons, Win95-style bevels) and Hi-Def (flat modern styling: system fonts, vector icons, no bevels). Both themes render the exact same pages — switching just swaps the stylesheet and icon set, instantly, without losing your place.

Navigation

Ordinary links between local .html pages (like the ones on this page) are intercepted and resolved relative to the page they appear on, then loaded the same way as the start page — so back, forward, and the home button all work uniformly, no matter how many pages deep you are.
