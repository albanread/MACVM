MACVM Architecture

MACVM is a research virtual machine for macOS on Apple Silicon, in the Self → Strongtalk lineage: a class-based object model with an adaptive optimizing compiler driven by type feedback.

It is not a port of either VM. It takes the adaptive-optimization machinery both share — inline caches, polymorphic inline caches (PICs), type feedback, and deoptimization — and combines that with Strongtalk's representation: classes with direct tagged pointers and no object table, rather than Self's prototype/map model.

Goals

  • Class-based object model — Strongtalk-style classes, direct tagged pointers, a two-word [mark][klass] header, and no object table.
  • Adaptive optimization — start executing quickly with a baseline interpreter, then recompile hot code using type feedback gathered from polymorphic inline caches, the technique Self pioneered and Strongtalk and HotSpot both inherited.
  • Apple Silicon first — arm64 calling convention, W^X / JIT hardening (MAP_JIT, pthread_jit_write_protect_np), and pointer-authentication awareness, rather than a portable-first design retrofitted for arm64.
  • Native macOS integration — this GUI shell is part of that: a direct objc_msgSend bridge to Cocoa/AppKit/WebKit, the same approach already proven in sibling projects, rather than a cross-platform toolkit.

Where things live

  • src/oops.rs — the object model: tagged pointers, headers, classes.
  • src/memory.rs — object memory, allocation, the garbage collector.
  • src/lookup.rs — method lookup, inline caches, PICs, type feedback.
  • src/interpreter.rs — the threaded-code interpreter (the baseline tier).
  • src/compiler/ — the adaptive optimizing compiler and its abstract codegen backend.
  • src/runtime.rs — stacks, activation frames, deoptimization.
  • world/ — the object world / image sources.
  • gui/ — this GUI shell: plan, reference pages, CSS — the very tree these pages live in.
  • docs/ — the full design record: SPEC.md, SPRINTS.md, DESIGN.md, and more.
