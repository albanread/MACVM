# S24 B5 Design — Multi-Basic-Block Block Splicing (conditional control flow, conditional NLR)

## 0. The B4 premise — verified, and its exact boundary

**Premise holds.** I re-read the materializer directly (not just the readers): the B4 arm at `src/runtime/deopt.rs:533-565` keys **solely** on `vf.scope.is_block && prev_fp != ENTRY_FRAME_SENTINEL` — it never inspects the resume bci. For every spliced `is_block` frame it (a) aliases FP+3 to the ROOT frame's Context and (b) calls `materialize_closure` (deopt.rs:154-177) to synthesize a **real closure** (`home_ref` = root fp+serial, `copied[0]` = root receiver, `copied[1]` = root Context iff `captures_ctx`) into the **receiver-arg slot** — exactly the slot `OP_NLR_TOS` reads. So an interpreted `nlr_tos` reached after a deopt at *any* bci of the block — including inside a conditional arm — delivers through a live home_ref. Additionally, DeoptRaw entries are self-contained (own bci + own stack snapshot + reexecute flag; deopt-bci reader), and the materializer's M5 resume-stack-height model already follows branch directions (the JumpFwd/BrTrueFwd arm-position walk, deopt.rs ~190-205) because **root methods deopt at in-branch bcis today**.

**The boundary:** B4 covers frame *reconstruction* at arbitrary bcis. What B5 must newly get right is compiler-side only: computing a *correct per-BB stack snapshot* to put into each DeoptRaw (today's linear walk gets it free from the single path), and the join/exit plumbing. No runtime, materializer, or metadata-encoding change is needed — see §5.

## 1. Hazards (risk-first, concrete failure scenarios)

**H1 — Entry-stack consistency at block-internal branch targets.** inputsKnown:'s block has joins BB4 (inner `or:`) and BB6 (outer `or:` → the `ifFalse:` seat), each entry depth 1 with a different vreg holding the arm's result per predecessor. If a join inherits one predecessor's exit stack verbatim, the other path reads the wrong vreg: `allKnown:` answers arm-2's truth value for an arm-3 element — **silent wrong answer**, no crash. Mitigation: reuse `compute_entry_depths` (ir.rs:695) + `entry_stack_sources` (ir.rs:753) + `emit_merges` (ir.rs:4905) verbatim — Merge blocks get fresh merge vregs, every predecessor Moves before jumping. The depth-table prerequisite already exists: NlrTos/BlockReturnTos deltas were added in S24 A1 (ir.rs:670-676).

**H2 — Multi-exit join discipline.** Three exit kinds with three destinies: fall-out value exits (BlockReturnTos/ReturnTos/ReturnSelf) → `Move{result_vreg}` + `Jump{continuation}`; NlrTos → `Ir::Ret` of home M, **no join edge**; cold-send traps → terminating `UncommonTrap`, no join edge. Failure if conflated: NlrTos jumping to the continuation makes T1's Finder both return x from `firstAbove:in:` *and* keep iterating (double-execution wrong answer); a trap arm feeding the join reads an undefined result. Per-return-fresh vregs instead of one shared `result_vreg` would make the continuation read whichever arm was translated last. The shared multi-def `result_vreg` is the codebase's proven join: `copy_propagate` never folds multi-def Moves (ir.rs:4989-4991) and S12 spill-all gives it one canonical frame slot.

**H3 — Value-send result when the NLR arm produces none / no arm returns.** BB7 never defines `result_vreg` — fine, control left M. Degenerate case: a block where **every** path NLRs or traps (`[:x | x ifTrue:[^1] ifFalse:[^2]]`) — the continuation has zero predecessors. The consumer must still adopt it (convert resumes M's remaining bytecode there, dead but well-formed); it must not feed a live merge or trip an entry-stack assert ("only real merge points are consulted", ir.rs:5640-5643 already states the split-block contract). Track `any_return` and debug-assert the shape.

**H4 — Deopt scopes in non-first BBs.** Verified sound by construction: the one shared `inline_proto` is legal across BBs because every field in it is extent-constant — `sender_bci` and `caller_pending_stack` are host-side facts fixed at the value-send, invariant under the block's internal control flow; only DeoptRaw.{stack,bci} vary per site, and per-BB stacks come from the same entry-stack machinery try_inline_cfg's in-body sites already trust (ir.rs:2627-2666). Concrete failure this discipline prevents: a deopt at BB2's cold `#determinedBy` send with a linear-model stack would rebuild the frame missing the or:-chain intermediate → interpreter resumes at bci 21 popping garbage. Reexecute:true snapshots inside an arm use that arm's local sim stack — exact per-BB.

**H5 — Stale-alias predicate under-approximation (escape.rs).** `site_has_sends.insert(bci, !is_leaf(block))` (escape.rs:562, verified) keys "does the splice introduce safepoints" on sends. A multi-BB body introduces safepoints **without** sends — every BoolBr's not_bool trap. Failure: a phantom alias live across the splice is recorded dead at the not_bool trap → deopt there rebuilds the frame with a reclaimed/stale slot → GC-dependent wrong value or crash. Fix: widen to `!is_leaf(block) || decode(block).blocks.len() > 1`.

**H6 — Blockarg recursion (flagship path): try_inline_cfg's 1:1 ccfg-block→IrBlock invariant.** The depth-3 recursion at ir.rs:2483 appends flat into the callee block's accumulating `bcode`; a multi-BB block cannot append flat. If we naively split, `exit_stacks[b]` / Inherit successors (ir.rs:2300-2302) and `emit_merges` still read the **pre-split** segment → successors get wrong entry vregs. Fix is cur_id threading (§3, step 5): entries into callee block b still target `ir_ids[b]` (predecessor jumps unaffected), but the terminator, exit stack, and merge-moves are keyed to the **final segment**.

**H7 — Budget blowup.** `inline_cost` = raw bytecode_len; the flagship block is 44 bytes (> the hardcoded level-1 per_call_cost 30 at escape.rs:519-523), and the Scale-customized compile splices it at **four** value: sites (~231 logical bytes + a trap block per BoolBr). `total_bytes` is declared but load-bearing nowhere (machinery reader's honest finding). Failure: N sites × 60-byte branchy blocks multiplies code size and compile time unbounded. Fix: block-side 2× bonus (44 ≤ 60) **plus** actually enforcing `total_bytes` via a cumulative counter on the Translator.

**H8 — bstack vs M-stack discipline.** The block's per-BB sim stacks start at depth 0 (blocks enter with an empty operand stack — matches `compute_entry_depths` seeding); M's operands are popped in lockstep with `stack_sites` *before* the graft and live only in `caller_pending_stack`. The one subtle spot is the branch-condition pop ordering (trap snapshot **includes** the condition, then pop for successor depth — the ir.rs:2118-2124 discipline); inverted, join depths go off-by-one → merge-vreg count mismatch → misaligned merge or assert. Reusing try_inline_cfg's Branch arm gets this right by construction.

**H9 — Dead BBs.** The flagship block's BB8 (frontend's join-jump after a dissolved `^`) is unreachable — no entry state ever computed for it. Must get the `reachable[]` inert-RetSelf placeholder treatment (ir.rs:2286-2294), never feed a merge. Template already handles this.

**H10 — regalloc/emit of `Ir::Ret` mid-CFG.** B4 exercised NLR Rets only in straight-line splices; now a Ret sits in a sub-block while merge vregs are live on sibling paths that skip it. Under spill-all this should be benign, but it's the one machinery-reader-flagged open risk — pinned by T3 (both exits hot in one nmethod) under DEOPT_STRESS/GC_STRESS.

**H11 — Loops in blocks.** A back-edge needs `Ir::Poll` + LoopPoll in an is_block scope — machinery exists (items-8 reuse) but is a novel deopt shape with zero payoff: **no target shape** (flagship, includes:/detect:ifNone:/anySatisfy:/allSatisfy:/Bag>>=) contains a loop. Decision: **decline back-edges in v1** to keep the DEOPT_STRESS verification matrix minimal; lift later behind the same relaxed predicate.

## 2. Approach: (b) — try_inline_cfg as the shared multi-BB graft engine, parameterized for block mode

Adopting the machinery reader's recommendation, with concurrence: try_inline_cfg (ir.rs:2136-2792) already contains **everything structurally hard** — CFG graft with fresh-id remap, Empty/Inherit/Merge entry stacks, merge Moves, the pre-allocated `result_vreg` + `continuation_id` join, per-path "ends without reaching the join" mechanics (which is *exactly* conditional NLR's shape: `Ir::Ret` on one arm, join on the other), per-extent InlineSite protos with parent chaining, InlineSite-chained not_bool traps with the branch-condition pop fix, and dead-block placeholders. Block mode adds only: four opcode arms lifted from translate_spliced_block_body (PushSelf→block_self; PushCtxTemp/StoreCtxTempPop depth-0 → ctx_vregs/method_ctx_vreg routing, ir.rs:4712-4748; NlrTos→`Ir::Ret`+block-ends; BlockReturnTos→the ReturnTos join), a proto override (is_block:true, receiver=home self, optional parent), guard elision (direct value receiver is statically the literal block — splice_block is guard-free today, ir.rs:4226-4228), and a parameterized eligibility recheck (is_inline_eligible_cfg rejects is_block/ctx-temps/NLR at inline.rs:173-186 and is re-checked at ir.rs:2155 — the gate must be **parameterized, not bypassed**). Literals/ICs already read the owning method's tables (`callee.literals().at(idx)`, `InterpreterIc::at(callee, ic)`) so block bytecode needs no re-encoding. Alternatives rejected in §8.

**Phasing:** Phase 1 = multi-BB blocks at **direct value sends** in M (consumer 1 — the continuation/split protocol already exists end-to-end: translate_site_instr may return `Some(Some(cont))`, convert consumes it at ir.rs:5626-5649). Phase 2 = multi-BB blocks at **in-callee value: sites** (consumer 2 — the flagship's actual path through `do:`), which requires the H6 cur_id threading.

## 3. The exact translation algorithm

New `graft_block_cfg` (block mode of the shared engine), inputs: block MethodOop `blk`, `blk_pool_ix`, operand vregs (value-send args), `block_self` (= `self_vreg` for consumer 1; root's self for consumer 2), the host's accumulating `code`/`deopt` context, `parent: Option<InlineSite>`.

1. `cfg = decode::decode(blk)`; **re-validate** the relaxed shape (assert backstop mirroring ir.rs:4293-4305 — operands already popped, no clean fallback).
2. Mint `ir_ids: Vec<BlockId>` (one fresh id per ccfg block), one `continuation_id`, one `result_vreg = fresh(true)`.
3. Slots: args **alias** the value-send operand vregs; temps = fresh nil-initialized vregs emitted in the host's current block (dominate the extent) — unchanged from splice_block ir.rs:4309-4321 / blockarg ir.rs:2459-2472.
4. Build `inline_proto` **once**: `{method_pool_ix: blk_pool_ix, receiver: block_self, slots, sender_bci: value-send bci in host scope, caller_pending_stack: host stack below the popped operands, is_block: true, parent}`. Cloned into every in-body safepoint.
5. `compute_entry_depths(cfg)` — entry depth 0 at block 0 (blocks enter stack-empty); the join-consistency assert (ir.rs:708-713) runs on block bytecode.
6. `entry_stack_sources`: Empty / Inherit(pred, exit stack verbatim) / Merge (fresh merge vregs sized to entry depth).
7. `reachable[]` pass: unreachable blocks (flagship BB8) → inert RetSelf placeholders.
8. Per reachable ccfg block, seed cstack from its entry source and walk with the **block arm set**: block-table literal/IC reads; the existing send ladder (smi/array fuses with `fresh_inlined_trap_block(proto)`, hot CallSend with reexecute:false below-send scope, cold send → terminating UncommonTrap with reexecute:true, block `trapped`); self-send static-klass devirt when `block_self == self_vreg`; `ReturnTos|BlockReturnTos` → `Move{result_vreg, tos}; Jump{continuation_id}`, returned=true; `ReturnSelf` → same with block_self; **NlrTos → `Ir::Ret{tos}`**, block ends, no join edge, bump `spliced_nlr`; terminator Jump → `emit_merges` (if Merge target) + `Jump{ir_ids[t]}`; terminator Branch → non-fused `BoolBr` with InlineSite-chained not_bool trap, condition popped **after** the trap snapshot; back-edge → unreachable!() (v1 gate excludes it). Record `exit_stacks[b]` from the block's final state for Inherit successors.
9. Return `(result_vreg, entry_id = ir_ids[0], continuation_id, any_return)`.

**bci/block-id mapping:** ccfg index → `ir_ids[i]`; every in-splice `DeoptRaw.bci` is **block-relative** (the block's own bytecode, interpreter-resumable at any bci per §0); host bcis appear only in `proto.sender_bci`. No bytecode rewriting anywhere.

**Consumer 1 adoption** (splice_block/translate_site_instr): push `result_vreg` + `stack_sites` in lockstep, arm `pending_jump_target = entry_id`, return `Some(Some(continuation_id))` — convert's root loop already does the rest.

**Consumer 2 adoption** (phase 2, inside try_inline_cfg's callee walk): thread `cur_id` per callee block (initially `ir_ids[b]`); at a multi-BB value: site, finish the current segment with `Jump{entry_id}`, set `cur_id = continuation_id`, continue appending the callee's remaining bytecode into cur_id's vec; `exit_stacks[b]`, terminator emission, and emit_merges all key to the **final segment**. Predecessor jumps into block b are unaffected (entry stays `ir_ids[b]`).

## 8. Rejected alternatives

**(a) Generalize translate_spliced_block_body into a second mini-CFG walk:** re-implements ~500 lines of the subtlest scaffolding (entry depths, merges, branch-condition pop accounting) that must agree with the root loop *by construction* — the ir.rs:2118-2124 fix exists precisely because two walks once disagreed. (a) and (b) converge on the same code; keep try_inline_cfg's copy. **(c) Bytecode pre-pass rewriting the block into M's stream:** destroys the two-level bci mapping deopt depends on (DeoptRaw.bci is callee-relative; a merged stream forces a fragile bci side-table), requires re-encoding literals/ICs into M's tables (breaking IC feedback), temp renumbering, branch re-encoding with overflow handling — a wholly new mechanism with zero reuse of proven deopt machinery. **Also rejected:** loops-in-blocks v1 (novel LoopPoll-in-is_block deopt shape, zero payoff); per-return fresh result vregs + a phi pass (no phi machinery exists; multi-def + spill-all *is* the repo's join discipline); klass-guarding the direct value splice (receiver is statically the literal block; guard-free today).

## 9. Soundness paragraph

Every interpreter-visible entry into a multi-BB spliced extent is a deopt safepoint carrying a self-contained DeoptRaw — its own block-relative bci, its own per-path-exact operand-stack snapshot (computed by the same entry-depth/merge machinery whose correctness the root method and method inlining already depend on, with frontend depth-consistency asserted at every join), and one is_block InlineSite whose every field is provably constant across the extent because `sender_bci` and `caller_pending_stack` describe the host's state at the value-send, which the block's internal control flow cannot alter. On any such deopt — in a condition BB, a taken arm, or the NLR arm itself — the materializer rebuilds the chain outermost-first and its B4 arm, conditioned only on `scope.is_block`, plants a real closure with `home_ref` = the just-materialized root's fp+serial into the receiver-arg slot; therefore an interpreted `nlr_tos` reached from any resume point delivers through a live home exactly as an ordinary activation would, and no `ensure:`/`ifCurtailed:` marker can sit interior to the materialized chain (markers land only on interpreter activations of protected blocks, which are primitive-excluded from inlining and splicing, and a protected block lexically containing an NLR block already fails eligibility). In compiled execution, `nlr_tos` lowers to `Ir::Ret` of the home compilation — correct because the home is by construction the compilation root — while value exits converge by Move into one shared, spill-all-canonicalized result vreg that copy-propagation cannot fold, and paths that never reach the join (NLR, traps) never define it and never read it, since control has left M or the compilation entirely. Hence every path through the spliced CFG is either (i) a value exit indistinguishable at the join from a called block's return, (ii) an NLR indistinguishable from the home method's own return, or (iii) a safepoint from which the interpreter resumes in a frame bit-identical to one the interpreter itself would have built — and an adversary must exhibit a fourth kind of path to refute this, which the accepted opcode/terminator set structurally excludes.
